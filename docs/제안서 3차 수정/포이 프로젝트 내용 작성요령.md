## **📋 권장 작성 범위**

### **각 프로젝트당 3-4줄 (약 80-120자)**
- 1줄: 프로젝트명 + 간단한 배경
- 1줄: 구조화한 데이터 유형 및 규모  
- 1줄: 생성형 AI 적용 방식/기술
- 1줄: 주요 성과/결과

### **전체 분량: A4 기준 1/3~1/2 페이지**

---

## **🎯 포함할 핵심 내용 템플릿**

### **프로젝트 1: [프로젝트명]**
- **배경**: [어떤 도메인/고객 대상인지]
- **데이터 구조화**: [○○ 데이터 ○○건을 ○○ 방식으로 구조화]
- **AI 적용**: [RAG/Knowledge Graph/기타 기술 활용]
- **성과**: [정량적 결과 또는 개선 효과]

---

## **💡 강조할 핵심 요소들**

### **1. 데이터 구조화 역량**
- 구조화한 데이터 **유형** (텍스트, 문서, 법령 등)
- 처리한 데이터 **규모** (○○만건, ○○GB 등)
- 사용한 **구조화 방식** (Knowledge Graph, 온톨로지 등)

### **2. 생성형 AI 기술력**
- **적용 기술** (RAG, Fine-tuning, Prompt Engineering 등)
- **활용 모델** (GPT, Claude, 자체 모델 등)
- **구현 방식** (API 연동, 자체 구축 등)

### **3. 프로젝트 성과**
- **정량적 결과** (정확도 ○% 향상, 처리시간 ○% 단축 등)
- **사용자 반응** (만족도, 사용률 등)
- **비즈니스 임팩트** (비용 절감, 효율성 증대 등)

### **4. 도메인 관련성**
- **법률/공공 분야** 경험 (있다면 강조)
- **복잡한 규칙 기반** 데이터 처리 경험
- **전문가 지식** 구조화 경험

## **📝 예시 구조**

```
### 6.2 주요 프로젝트 실적

**1) ○○ 법률정보 통합 플랫폼 구축**
- ○○기관 대상, 법령 및 판례 데이터 10만건 Knowledge Graph 구조화
- GPT-4 기반 RAG 시스템으로 법률 Q&A 서비스 구현
- 검색 정확도 85% 달성, 답변 생성 시간 70% 단축

**2) ○○ 기업 내부 문서 지식화 프로젝트**
- 비정형 업무 문서 5만건을 구조화된 지식베이스로 전환
- 맥락 기반 문서 검색 및 자동 요약 시스템 개발
- 직원 업무 효율성 40% 향상, 문서 검색 시간 80% 단축
```

**핵심은 구체적 수치와 기술적 신뢰성을 보여주면서도 간결함을 유지하는 것**입니다. 법률구조공단 입장에서 "이 회사가 우리 프로젝트를 성공적으로 수행할 수 있을까?"라는 의구심을 해소하는 것이 목표입니다.
